# Счёт-оферта: что заполнять в печатной форме

Правовая природа: **оферта** (ст. 435 ГК РФ) — предложение заключить договор; **акцепт** — обычно полная оплата по счёту (ст. 438, 443 ГК РФ), если иное не указано.

## Обязательные блоки в макете

| Блок макета | Содержание | Источник в расширении |
|-------------|------------|------------------------|
| **ЗаголовокСчета** | Банк получателя, ИНН, КПП, БИК, р/с, к/с, получатель | `ПолучитьТаблицуСведенийСчетаНаОплату` + банковский счёт из счёта |
| **Заголовок** | «СЧЕТ-ОФЕРТА № … от …» | Номер и дата счёта |
| **Поставщик** | Наименование, ИНН, КПП, адрес, телефоны продавца | `ОписаниеОрганизации` |
| **Покупатель** | То же для покупателя + строка «Оферта» | Фиксированный текст в `ГСК5664_ТекстОснованияОферты` (без договора контрагента) |
| **Таблица** | Наименование, кол-во, цена, % НДС, сумма НДС, сумма | ТЧ «Товары» счёта |
| **Итого** | Сумма к оплате | Итог по таблице |
| **ДополнительныеУсловия** | Текст оферты, срок акцепта, существенные условия | Справочник «Дополнительные условия» + фиксированный текст в `ГСК5664_ТекстУсловийОферты` |

Макет для пользовательского редактирования: `Документ.СчетНаОплатуПокупателю.ПФ_MXL_СчетОферта` (узел «Счет покупателю» в «Макеты печатных форм»).

## Заголовок банка (два варианта в макете)

| Область | Когда выводится |
|---------|-----------------|
| **ЗаголовокСчета** | Нет логотипа организации (или нет области с логотипом) |
| **ЗаголовокСчетаСЛоготипом** | В карточке организации задан **ФайлЛоготип** |

QR-код — для физлица-покупателя, рубли, как в типовом счёте БП. Область **«Шапка»** — служебная строка макета (как в `ПФ_MXL_СчетЗаказ`), в печать не выводится.

## НДС и подписи

В таблице выводятся колонки **%** (ставка НДС) и **Сумма НДС** по каждой строке; итоговая строка НДС — как в типовом счёте БП (**ИтогоНДС**, **ИтогоКОплате**, **СуммаПрописью**, **Подвал**).
